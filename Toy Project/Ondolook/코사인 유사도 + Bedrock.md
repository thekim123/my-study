너무 좋다 이거. 진짜 실전적인 구조야.  
**코디 추천을 위한 "기상 + 일정" 기반 코사인 유사도 매칭 → 베드락 리파인 요청**  
이거 완전 추천 알고리즘 설계의 정석적인 흐름이야.

---

## 🔥 구조 다시 요약하면

### 📥 입력

- 날씨 정보 (맑음, 흐림 등 텍스트 or 수치형 변환)
    
- 온도, 강수량, 미세먼지, 자외선 (수치형 입력)
    
- 일정 카테고리 (일상, 비즈니스, 데이트 등)
    

---

### ⚙️ 1. **코사인 유사도 기반 후보 선정**

- 사전에 만들어둔 코디 에셋들에 대해 vectorization
    
    ```json
    {
      "weather": "맑음",
      "temp": 28,
      "pm10": 30,
      "uv": 8,
      "category": "데이트"
    }
    ```
    
    → 이걸 전처리해서 `[맑음:1, 흐림:0, ...], temp: 28, pm10: 30, uv: 8, category: 데이트` 이런 벡터로 변환  
    → 에셋도 동일하게 vector로 구성해놓고
    
- 사용자의 현재 조건을 벡터화 → 사전 에셋들과 **코사인 유사도 계산**
    
- 가장 가까운 상위 3~5개 정도 선택
    

---

### 🧠 2. **Bedrock으로 refinement (옵션)**

- 선택된 후보들을 기반으로 베드락에 프롬프트 요청:
    
    > "이런 날씨와 일정 조건에서 이 코디 중 가장 적합한 걸 골라줘"  
    > → 또는 메시지/설명 생성:  
    > "덥고 맑은 날씨엔 밝은 색상의 린넨 셔츠가 잘 어울려요!"
    

### ➕ 또는 최종 선택 없이

- 베드락에게 직접 물어볼 수도 있어:
    
    > `"맑고 더운 여름, 데이트 일정에 어울리는 캐주얼한 코디를 골라줘"`
    

---

### 🧊 3. 캐싱 최적화 (나중에)

- 여름 맑은 날, 데이트 일정 → 자주 나오는 조합
    
- 미리 베드락 응답 저장 → 캐싱
    
- 자주 등장하는 weather+category 조합 위주로 캐싱
    

---

## ✅ 장점

|항목|설명|
|---|---|
|빠른 반응성|코사인 유사도는 미리 계산된 벡터와 비교만 하면 됨|
|AI 비용 최소화|필요한 경우에만 Bedrock 요청|
|튜닝 유연성|유사도 기준, 가중치 등 쉽게 조정 가능|
|캐싱 가능성|자주 등장하는 날씨+일정 조합은 미리 뽑아둘 수 있음|

## 🧠 보너스: 벡터 구조 예시

```json
{
  "맑음": 1, "흐림": 0, "비": 0,
  "temp": 30,
  "rain": 0,
  "pm10": 20,
  "uv": 8,
  "일상": 0, "비즈니스": 0, "데이트": 1, "운동": 0
}
```

> 이렇게 만든 뒤 `코디 에셋`도 전부 이 벡터 형태로 저장해두고, 사용자 벡터와 코사인 유사도로 비교하면 끝.
