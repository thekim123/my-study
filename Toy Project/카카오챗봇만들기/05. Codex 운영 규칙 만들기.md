# AGENTS.md (한국어 번역)
## 프로젝트 미션
다음 요소들을 이용해 **프로덕션 수준 HR 리뷰 코멘트 추천 시스템**을 구축한다.
- `evaluation_case_snapshot`
- 하이브리드 검색 (유사 사례 / 정책 용어 / 스타일 예시)
- OpenAI **Responses API + Structured Outputs**
- validators
- human approval
- **SFT 먼저, 이후 DPO**

이 저장소는 **데모용 샌드박스가 아니다.**
모든 변경은 반드시 다음을 유지해야 한다.
- 보안
- 추적 가능성
- 감사 가능성 (auditability)

---
# 절대적인 제품 규칙 (Non-negotiable rules)
1. **Snapshot 계약을 우회하지 않는다.**  
    모델에 직접 raw relational join 결과를 넣지 않는다.
2. **스코프를 섞지 않는다.**  
    Public / Private 리뷰 경로는 **데이터 접근 계층에서 완전히 분리**되어야 한다.
3. **Writer만 있는 로직을 배포하지 않는다.**  
    추천 생성은 반드시 **validator와 함께 존재해야 한다.**
4. **승인되지 않은 데이터로 학습하지 않는다.**  
    SFT/DPO 데이터셋은 **큐레이션된 승인된 export**에서만 만들어야 한다.
5. **감사 가능성을 약화시키지 않는다.**  
    모든 추천은 다음을 통해 추적 가능해야 한다.
    - snapshot version
    - retrieval bundle
    - prompt version
    - model version
    - reviewer action
        
6. **OpenAI payload를 무분별하게 늘리지 않는다.**  
    모델에는 **최소한의 context만 전달한다.**

---

# OpenAI 사용 규칙
제품 추천 생성 호출은 **Responses API**를 사용한다.
추천 생성 계약은 반드시 **Structured Outputs**를 사용한다.
기본은 **stateless request**이며 `store=false`를 설정한다.
가능한 경우 `safety_identifier` 해시를 붙인다.
**HR 원본 데이터셋을 외부 hosted retrieval에 업로드하지 않는다.**
파인튜닝 데이터 export는 다음 조건을 만족해야 한다.
- version 관리
- 승인됨
- 비식별화
- 학습용으로만 업로드
- 정책에 따라 학습 후 삭제

# 보안 및 프라이버시 규칙
모든 HR 데이터는 **민감 정보로 취급한다**
다음 행동 금지
- reviewer 코멘트 원문을
    - 문서
    - 테스트
    - 스크린샷
    - 이슈
    
    에 복사하는 것

테스트에는 **비식별 fixture** 사용

public/private 안전성은  
**prompt가 아니라 코드로 강제한다**

public 추천 경로에서는  
**private evidence ref 반환 금지**

다음 위치에 secret 저장 금지
- 코드
- 테스트
- 노트북
- 샘플 config

# Retrieval 규칙
Retrieval bundle은 반드시 **3개로 분리**

1️⃣ similar_cases  
2️⃣ policy_terms  
3️⃣ style_exemplars

모든 retrieved item은 다음 필드를 포함해야 한다.
- stable source id
- scope
- quality score
- evidence ref / provenance

---

# 추천 출력 계약
추천 생성은 반드시 **structured JSON**을 반환한다.

최소 필드
```
scope
drafts[]
drafts[].text
drafts[].required_terms_used
drafts[].evidence_refs
drafts[].risk_flags
reasoning_summary
```

모델이 schema를 만족하지 못하면
→ **fail closed**  
→ typed error 반환
