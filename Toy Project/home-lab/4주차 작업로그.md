## 4주차: 홈랩 배포 + 문서화 + 데모화
이번 주 목표는 **“내 컴퓨터에서만 되던 것”을 끝내는 것**이다.

### 할 일

1. **배포 형태 최소화**
    - Docker 하나 또는 compose 하나
    - FastAPI + 간단한 web/CLI
    - 환경변수 분리
    - 로그 파일 분리
        
2. **홈랩 반영**
    - Proxmox 위에 VM 또는 LXC 하나
    - 거기에 앱 올리기
    - 내부망에서 접속 확인
    - 재부팅 후 자동 복구 확인
        
3. **운영 체크**
    - 인덱스 재생성 방법 문서화
    - 데이터 폴더 백업 경로 정리
    - API key 관리 방식 정리
    - 장애났을 때 재시작 절차 정리
        
4. **포트폴리오화**
    - README 정리
    - 아키텍처 그림 1장
    - 데모 GIF 또는 짧은 영상
    - “왜 이런 구조를 선택했는지” 작성
        

### 주말까지 나와야 하는 결과
- 홈랩에서 접근 가능한 실행본
- README
- 데모 자료
- 회고 문서
    

### 여기서 중요한 것
홈랩은 **앱을 띄우기 위한 수단**이어야지,  
**클러스터/HA/Ceph 놀이**로 가면 안 된다.

Proxmox 자체는 KVM과 LXC를 한 인터페이스로 다루기 좋기 때문에 홈랩 베이스로는 충분하다. 하지만 이번 4주의 목적은 인프라 연구가 아니라 **배포 완료**다. ([Proxmox](https://www.proxmox.com/en/products/proxmox-virtual-environment/overview?utm_source=chatgpt.com "Proxmox Virtual Environment"))
