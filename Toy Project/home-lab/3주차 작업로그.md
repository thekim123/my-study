## 3주차: tracing + eval + 품질 개선
이번 주 목표는 **“대충 잘 되는 느낌”을 버리고, 어디서 망하는지 보이게 만드는 것”**이다.

LangSmith 쪽 공식 문서도 tracing에서 retrieval step과 LLM call을 함께 보고, evaluation에서는 dataset·target function·evaluators 구조로 점검하는 흐름을 제시한다. 이번 주는 그걸 그대로 네 프로젝트에 적용하면 된다. ([LangChain Docs](https://docs.langchain.com/langsmith/observability-quickstart?utm_source=chatgpt.com "Tracing quickstart - Docs by LangChain"))

### 할 일
1. **LangSmith tracing 붙이기**
    - 질문 1회마다 trace 1개
    - 어떤 도구를 썼는지
    - 어떤 chunk를 참고했는지
    - 응답까지 얼마나 걸렸는지
        
2. **평가셋 20개 만든다**
    - 쉬운 질문 5개
    - 파일 위치 찾기 5개
    - 여러 문서를 합쳐야 하는 질문 5개
    - 답이 없어야 하는 질문 5개
        
3. **실패 유형 분류**
    - retrieval miss
    - chunking 문제
    - tool selection 문제
    - prompt 문제
    - answer formatting 문제
        
4. **한 번에 하나씩 고친다**
    - chunk size 바꾸기
    - top-k 조정
    - rerank 여부 점검
    - system prompt 정리
    - “모르면 모른다” 기준 강화
        

### 주말까지 나와야 하는 결과
- 평가 질문셋 20개
- baseline 결과
- 개선 후 결과
- “뭐가 문제였고 어떻게 고쳤는지” 한 페이지 정리
