### 1. 자바 프로젝트에서 넣을 파일 확정

우선 포함 파일 타입을 정해.

처음엔 이 정도만 추천한다.

- `.java`
- `.md`
- `.txt`
- `.yml` / `.yaml`
- `.properties`

선택적으로:
- `build.gradle`, `settings.gradle`
- `pom.xml`

처음부터 넣지 말 것:
- `target/`, `build/`
- `.class`
- 이미지
- 테스트 리포트
- 로그
- generated code

즉 **인증 흐름을 설명하는 데 실제로 쓰이는 코드와 문서만** 넣으면 된다.

---

### 2. 파이썬으로 파일 로더 작성
여기서 1차 목표는 임베딩이 아니라 **파일을 잘 긁어오는 것**이다.

해야 하는 일:
- 디렉토리 재귀 순회
- 포함 확장자만 허용
- 제외 디렉토리 필터링
- 파일 내용 읽기
- path, content, extension 저장

데이터 구조는 이 정도면 충분하다.
```python
from dataclasses import dataclass

@dataclass
class SourceDocument:
    path: str
    content: str
    file_type: str
```

완료 기준은 콘솔에 이런 게 찍히는 상태다.
```text
[LOAD] src/main/java/com/example/auth/AuthService.java
[LOAD] src/main/java/com/example/auth/AuthController.java
[LOAD] docs/login-flow.md
Loaded documents: 11
```

---

### 3. chunking 구현
그다음 문서를 chunk로 나눈다.

자바 파일이라고 해서 처음부터 AST 파싱할 필요 없다.  
이번 주는 **텍스트 청크**면 충분하다.

권장값:
- chunk size: 800~1200 chars
- overlap: 100~200 chars

구조 예시:
```python
@dataclass
class DocumentChunk:
    chunk_id: str
    source_path: str
    text: str
    start: int
    end: int
```
중요한 건 **각 chunk가 어느 파일에서 왔는지 반드시 유지**하는 것이다.

---

### 4. retrieval만 먼저 검증
여기서 바로 LLM 붙이지 말고 먼저 검색만 본다.

질문:
```text
이 프로젝트에서 인증 흐름이 어떻게 되어 있지?
```

그랬을 때 top-k가 대충 이런 식으로 나와야 한다.
```text
1. docs/login-flow.md
2. src/main/java/.../AuthService.java
3. src/main/java/.../JwtTokenProvider.java
4. src/main/java/.../AuthController.java
```
이게 맞으면 그 다음에 답변 생성을 붙이면 된다.

## 간단한 구현 순서

파이썬 프로젝트 폴더를 이렇게 잡아도 된다.

```text
rag_mvp/
  app.py
  loader.py
  chunker.py
  models.py
  indexer.py
  retriever.py
  data/
  logs/
```

그리고 순서대로:

- `models.py` → `SourceDocument`, `DocumentChunk`
- `loader.py` → 자바 프로젝트 파일 읽기
- `chunker.py` → 문자 기반 분할
- `app.py` → 실행해서 결과 출력
    

---

## 한 가지 실무적으로 중요한 점

자바 코드는 import, annotation, boilerplate가 많아서  
chunk가 너무 작으면 의미가 잘 끊긴다.  
그래서 처음엔 **조금 크게 자르는 편**이 낫다.

예:

- 1000자 전후
    
- overlap 150자 전후
