## 1주차: 범위 고정 + 로컬 MVP
이번 주 목표는 **“일단 돌아가는 것”**이다.

### 할 일
1. ✅**범위를 자른다**
    - ✅리포는 1개만
    - ✅문서 폴더도 1개만
    - ✅PDF까지 한꺼번에 넣지 말고, 우선 markdown/text/code 위주로 시작
    - ✅UI도 웹 말고 CLI 또는 아주 얇은 FastAPI 하나면 충분
        
2. ✅**기본 파이프라인 만든다**
    - ✅문서 로드
    - ✅chunking
    - ✅임베딩/인덱싱 - AWS Bedrock 사용함(프리티어 크레딧 사용).
    - ✅질문 → retrieval → 답변 생성
    - ✅답변에 source path 붙이기
        
3. ✅**답변 정책 정한다**
    - ✅모르면 모른다고 말하기 - **코사인 유사도가 0.5 이하이면 모름**
    - ✅근거 없으면 추측하지 않기 - **코사인 유사도가 0.5이하이면 근거없음**
    - ✅답변 끝에 참고한 파일 2~5개 보여주기
        
4. **실행 로그 남기기**
    - 질문
    - 검색된 문서 수
    - 선택된 chunk
    - 최종 응답
        

### 주말까지 나와야 하는 결과
```text
ask("이 프로젝트에서 인증 흐름이 어떻게 되어 있지?")
-> 답변
-> 참고 파일: auth/service.ts, docs/login-flow.md
```

### 이번 주 체크포인트
- 30~50개 문서 수준에서 돌아가는가
- 근거 파일이 실제로 맞는가
- 헛소리 비율이 높은가 낮은가

### 이번 주 금지
- 멀티에이전트
- 예쁜 프론트
- 복잡한 DB
- 홈랩 설정놀이
