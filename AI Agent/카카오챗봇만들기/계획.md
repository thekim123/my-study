## 1) AWS Bedrock 준비

1. AWS 콘솔 → **Bedrock** → **Model access**
    
    - 사용할 모델(예: Claude 계열, Llama 등) **Access 요청/활성화**
        
2. IAM 사용자/역할 생성
    
    - 권한: `bedrock:InvokeModel` (최소권한으로)
        
3. 로컬 또는 서버에 AWS 자격증명 세팅
    
    - 환경변수 또는 `~/.aws/credentials`
        
4. “Bedrock 단독 호출” 테스트
    
    - 간단한 스크립트로 `InvokeModel` 성공 확인 (여기서 막히면 뒤에 전부 못 감)
        

---

## 2) Webhook 서버(백엔드) 뼈대 만들기

5. 서버 프레임워크 선택
    
    - 빠르게: **FastAPI**(Python) / Express(Node) / Spring Boot(Java)
        
6. 엔드포인트 1개 만든다
    
    - `POST /kakao/webhook`
        
7. 카카오 챗봇 요청 JSON에서 사용자 발화(utterance) 파싱
    
8. Bedrock 호출해서 텍스트 응답 받기
    
9. 카카오 응답 포맷으로 리턴
    
    - `version: "2.0"`
        
    - `template.outputs[0].simpleText.text`
        

---

## 3) 외부에서 접근 가능하게 배포(HTTPS)

10. 개발 단계(가장 빠름)
    

- ngrok 또는 Cloudflare Tunnel로 `https://xxxxx` 만들기
    

11. 운영 단계(권장)
    

- EC2 or ECS(Fargate) + ALB + ACM(무료 TLS)
    
- 또는 API Gateway + Lambda(서버리스)

## 4) 카카오 채널/챗봇/스킬(Webhook) 연결

12. **카카오 채널 생성**
    

- 카카오 채널 관리자에서 채널 만들기
    

13. **카카오 i 챗봇 생성** 및 채널 연결
    
14. **스킬 생성(외부 API 호출)**
    

- 스킬 URL에 `https://.../kakao/webhook` 입력
    

15. 시나리오(블록)에서
    

- “사용자 발화 → 스킬 호출 → 응답 출력” 흐름 연결
    

16. 테스트 콘솔에서 대화 테스트
    

- 요청이 서버로 들어오는지 로그 확인
