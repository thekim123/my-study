## 1) IAM Role 만들기 (Lambda 실행 역할)

1. IAM → Roles → Create role
2. **Trusted entity type**: `AWS service`
3. **Use case**: `Lambda`
4. Permissions(정책) 연결
    - `AWSLambdaBasicExecutionRole` (필수)
        
5. Role 생성 후 **Inline policy 추가**(Bedrock 호출)
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "BedrockInvoke",
      "Effect": "Allow",
      "Action": ["bedrock:InvokeModel"],
      "Resource": "*"
    }
  ]
}
```

---

## 2) Lambda 함수 만들기 (Bedrock 호출 + 카카오 응답 포맷 반환)

6. Lambda → Create function
    - Runtime: Python 3.12
    - Execution role: 1)에서 만든 Role 선택
        
7. Environment variables(권장)
    - `BEDROCK_REGION`: 예 `us-east-1`
    - `MODEL_ID`: 예 `anthropic.claude-3-haiku-20240307-v1:0` (너 계정에서 보이는 모델로)
        
8. Lambda 코드 배포 (핵심)
    
    - `event["body"]` JSON 파싱
        
    - `userRequest.utterance` 읽기
        
    - Bedrock `converse()` 호출
        
    - 카카오 포맷 JSON을 **API Gateway 프록시 응답**으로 리턴
        

(이 단계에서 **Lambda Test**로 먼저 확인)  
9. Lambda 콘솔에서 Test event 만들어 실행

- 아래 같은 body로 테스트
    

```json
{
  "body": "{\"userRequest\":{\"utterance\":\"안녕\"}}"
}
```

- 성공하면 CloudWatch 로그에 응답/에러 확인 가능
    

---

## 3) API Gateway(HTTP API)로 HTTPS 엔드포인트 만들기

10. API Gateway → Create API → **HTTP API** 선택
    
11. Integrations: **Lambda function** 선택 → 2)에서 만든 함수 연결
    
12. Routes 생성
    

- `POST /kakao/webhook`
    
- Integration: 위 Lambda
    

13. Stages
    

- `default`(자동 생성) 또는 `prod`
    

14. Deploy
    
15. **Invoke URL** 확인
    

- 예: `https://xxxx.execute-api.us-east-1.amazonaws.com`
    

---

## 4) API Gateway에서 실제 호출 테스트 (가장 중요)

16. 로컬에서 curl로 테스트
    

```bash
curl -X POST "https://xxxx.execute-api.us-east-1.amazonaws.com/kakao/webhook" \
  -H "Content-Type: application/json" \
  -d '{"userRequest":{"utterance":"테스트"}}'
```

17. 기대 결과
    

- HTTP 200
    
- body에 카카오 포맷:
    

```json
{
  "version":"2.0",
  "template":{"outputs":[{"simpleText":{"text":"..."}}]}
}
```

## 5) (여기까지 되면) 카카오 챗봇 연결

18. 카카오 i 챗봇 관리자 → 스킬(외부 API)
    
19. URL에 위 Invoke URL + path 넣기
    

- `https://xxxx.execute-api.../kakao/webhook`
    

20. 시나리오에서 “사용자 발화 → 스킬 호출” 연결 후 테스트
    

---

## 흔한 장애 포인트 3개

- 401/403: Role에 `bedrock:InvokeModel` 누락
    
- 502: Lambda가 **프록시 응답 형식**(statusCode/body)으로 안 돌려줌
    
- 모델 관련 오류: `MODEL_ID`가 리전/권한과 불일치
