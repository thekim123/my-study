DevOps는 단순한 도구의 모음이 아니라 **문화 + 프로세스 + 기술**이 어우러진 개념이다.

## 🎯 단기 목표 (2025)

- **CI/CD 구현**
  - GitLab CI/CD 파이프라인 구성
  - 외장 Tomcat 대상 자동 배포 설정
  - GitLab Runner 연동
  
- **협업 도구 체계 구축**
  - ❌ Wiki.js를 문서화 도구로 사용 -> 이슈관리는 하기 힘듬
  - Mattermost로 커뮤니케이션 및 알림 관리
  - ❌ Mattermost에 있는 Focalboard를 이용하여 이슈관리를 할 수 있음. 
		  -> 알고보니 GitLab에 이슈관리 기능이 있음
		  -> GitLab 사용

  - GitLab 이슈 트래킹과 연동

- **알림 구성**
  - GitLab과 Mattermost를 연동하여 알림 수신 체계 구성
  - 추후 Incident Management 도구로 확장 고려 (ex. PagerDuty)

## ⚙️ 인프라 자동화 / 컨테이너화 계획

- **Ansible을 활용한 구성 자동화**
  - 직접 제어 가능한 인프라는 제한적이지만, 설정 자동화와 배포 편의성을 위해 구성 예정

- **Kubernetes 클러스터 활용 (8대 서버)**
  - AI 엔진 Pod는 NodeAffinity로 분산 (1~2개/노드)
  - HPA 도입으로 부하 시 자동 확장
  - `.dat` 파일 쓰기 충돌 방지: 쓰기 전용 Pod 1개 + 읽기 Pod N개 구조 설계
  - 이를 통해 **소프트 오토스케일링** 구현 *(조건부 수동 확장 + HPA의 결합 구조)*
