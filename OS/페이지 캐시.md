페이지 캐시(**Page Cache**)는 운영체제(OS)가 디스크로부터 읽어온 데이터를 메모리(RAM)에 임시 저장해두는 영역을 말합니다.

좀 더 쉽게 풀어보면,  
**"자주 사용하는 데이터를 매번 느린 디스크에서 읽는 대신, 빠른 메모리에 올려두고 다음 요청 때 더 빠르게 접근하기 위해 운영체제가 사용하는 캐시 영역"**입니다.

---

## 📌 페이지 캐시가 필요한 이유

- **디스크 접근 속도**는 메모리 접근 속도에 비해 매우 느립니다.
    
    - **RAM 접근 속도**: 약 **나노초(ns)** 단위
        
    - **SSD 접근 속도**: 약 **마이크로초(μs)** 단위
        
    - **HDD 접근 속도**: 약 **밀리초(ms)** 단위
        

메모리와 디스크의 접근 속도는 최소 수백배에서 수천배까지 차이가 나기 때문에, 매번 디스크에서 데이터를 읽으면 성능이 급격히 떨어집니다.

운영체제는 이를 개선하기 위해 디스크에서 읽어온 데이터를 RAM에 **페이지 단위(보통 4KB)** 로 임시 저장합니다. 이것이 바로 페이지 캐시입니다.

---

## 📌 페이지 캐시 작동 방식

예를 들어, 다음과 같은 과정이 발생합니다.

1. 애플리케이션이 디스크에서 특정 파일을 읽으려 요청
    
2. OS가 요청된 파일을 디스크에서 읽어서 페이지 캐시에 저장
    
3. 이후 애플리케이션이 동일 데이터를 다시 읽으려 하면,  
    디스크에 접근하지 않고 **페이지 캐시**에서 빠르게 반환
    

즉, 최초 접근만 디스크 접근이 발생하고, 이후 접근은 메모리로부터 이루어져 훨씬 빠릅니다.

---

## 📌 카프카와 페이지 캐시의 관계

카프카(Kafka)는 이 페이지 캐시를 적극적으로 활용하는 대표적인 서비스입니다.

- 카프카는 메시지를 파일로 저장(로그 세그먼트)합니다.
    
- 메시지를 읽거나 쓸 때 직접 메모리에 캐싱하지 않고, 대신 OS가 관리하는 **페이지 캐시**를 통해 빠르게 접근합니다.
    
- 그래서 카프카가 자체적으로 힙 메모리(heap memory)는 비교적 적게 쓰는 대신,  
    OS 페이지 캐시를 적극적으로 활용하여 성능을 높입니다.
    

이는 Kafka가 별도의 캐시 시스템을 직접 구현하지 않고, OS의 페이지 캐시 기능을 활용한 똑똑한 설계입니다.

---

## 📌 페이지 캐시와 리눅스 메모리 관리

리눅스에서 메모리 사용량을 확인하면, 종종 **"free memory가 매우 적다"** 는 상황이 발생합니다.  
이는 실제 메모리가 부족한 게 아니라, 운영체제가 **남는 메모리를 전부 페이지 캐시**로 활용하기 때문입니다.

리눅스에서는 RAM을 최대한 활용하기 위해, 여유 메모리를 자동으로 파일 캐싱(페이지 캐시)에 사용합니다.  
이 페이지 캐시는 다른 프로세스가 메모리를 요청하면 언제든지 다시 비워서 제공될 수 있는, 일종의 "**유연한 메모리 사용법**"입니다.

### 예시 (`free` 명령어 결과):

```bash
$ free -m
              total        used        free      shared  buff/cache   available
Mem:          16000        1500         500          50       14000       14000
```

여기서 실제 사용 가능(available)은 14GB나 되는데,  
`free`가 적게 나오는 건 **페이지 캐시(buff/cache)**로 14GB가 사용 중이기 때문입니다.

---

## 📌 페이지 캐시의 한계 및 고려할 점

- 페이지 캐시는 OS가 자동으로 관리합니다.
    
- 특정 프로세스(예: Kafka)가 너무 많은 페이지 캐시를 차지하면,  
    다른 애플리케이션의 성능에 악영향을 줄 수 있습니다.
    
- 그래서 Kafka와 같이 페이지 캐시를 많이 쓰는 서비스는 **별도 서버에 배포하는 게 권장**되는 것입니다.

## 📌 요약 정리

|구분|내용|
|---|---|
|정의|OS가 디스크 데이터를 RAM에 임시 저장해 성능을 높이는 캐시|
|단위|일반적으로 4KB 단위|
|관리 주체|운영체제 (Kernel)|
|카프카 활용|적극적으로 활용 (메시지 읽기/쓰기 속도 향상 목적)|
|리눅스 메모리 활용|리눅스가 남는 메모리를 모두 페이지 캐시로 사용|
|운영 고려사항|Kafka는 페이지 캐시 점유율이 높아 단독 서버 운영 권장|

즉, **페이지 캐시는 디스크 접근 속도를 개선해주는 핵심 메커니즘이며**, Kafka는 이를 매우 잘 활용하는 서비스라고 이해하면 됩니다.
