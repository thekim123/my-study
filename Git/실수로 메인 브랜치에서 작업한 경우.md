### 👉 "실수로 메인 브랜치에 직접 커밋해버렸고, 사실은 feat 브랜치에서 작업했어야 했던 경우"

=> 정답: **리셋(hard) 말고 `브랜치 분기 + 리셋 soft`나 `cherry-pick`, `rebase`를 사용하면 됨!**

---

## ✅ 시나리오 가정

지금 상태:

```
* a1b2c3d ← HEAD (main)  ← 실수로 커밋한 작업
* z9x8y7w               ← 원래 main HEAD
```

---

## 💡 해결방법 1: **브랜치 새로 만들고, main 되돌리기**

### 1️⃣ 현재 상태를 **새 브랜치로 분리**

```bash
git checkout -b feat/my-feature
```

👉 그러면 실수로 커밋한 작업은 `feat/my-feature` 브랜치에 안전하게 보존됨!

---

### 2️⃣ main 브랜치로 돌아가서 이전 커밋으로 되돌림

```bash
git checkout main
git reset --hard origin/main
```

- 이미 원격 `origin/main`에는 실수 커밋이 푸시되지 않았다는 전제야.
- 만약 푸시했다면 → `git push --force`가 필요 (주의!)

---

✅ **결과:**

- `feat/my-feature`: 작업한 커밋 보존됨
- `main`: 깔끔하게 원래 상태로 복구

---

## 💡 해결방법 2: 커밋을 임시로 stash + 브랜치 따기 (변경 중이라면)

```bash
git stash
git checkout -b feat/my-feature
git stash pop
```

이건 **아직 커밋도 안 한 상태에서 main에서 작업하고 있었던 경우** 유용함

---

## 📌 푸시까지 한 경우엔?

```bash
git push --force origin main
```

> 팀 프로젝트면 **반드시 팀원과 상의하고!**  
> 메인 브랜치에 `force push`는 신중하게 해야함.

---

## ✍️ 정리

|상황|해결 방법|
|---|---|
|실수로 main에 커밋만 했음 (푸시 X)|✅ 브랜치 새로 따고 → main을 reset|
|작업 중 (커밋 X)|✅ stash → 브랜치 → stash pop|
|main에 푸시까지 해버림|⚠️ 팀원과 상의 후 `force push` 가능|
|커밋 여러 개 쌓였을 경우|`rebase -i`, `cherry-pick` 활용 가능|

---
커밋만 한 상태고 푸시 안 했으면 **걱정하지 말고 브랜치로 빼두면 100% 복구 가능**
